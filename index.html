Assistant Flow

A small chat UI.
A strict safety path.

The assistant is intentionally simple on screen, but the path behind it is structured: message in, moderation first, backend risk check, then either a supportive reply or a crisis-support handoff.

💬
01
Qt Frontend

The user types into the desktop app. The message is sent from the assistant screen through QNetworkAccessManager to a local Express route.

🧪
02
Moderation First

The backend calls OpenAI Moderation on every message before normal generation. This makes safety a gate, not an afterthought.

🛡
03
Backend Risk Logic

Extra backend checks scan for self-harm, suicide, violence, abuse, and inability to stay safe. If triggered, normal coaching stops immediately.

🌿
04
Supportive Response

Safe messages go to the OpenAI Responses API with a student-wellness system prompt. High-risk messages return a hardcoded, compassionate crisis-support reply instead.

My Journal

Write it down.
Keep it yours.

Research consistently shows that regular journaling reduces stress, sharpens clarity, and builds resilience over time. MindEase makes the habit effortless.

  • ⏱
    Live Date & Time A live clock updates every second. Your entry is stamped with the exact moment automatically — no manual dating needed.
  • ✏️
    Guided Gratitude Prompts Four research-backed prompts are always visible. Use them as inspiration or ignore them — write whatever is on your mind.
  • 📁
    Saved to Your Own Files Entries save as .txt files in ~/Documents/MindEase_Journal/ — readable in any app, forever.
  • 📖
    Browse Past Entries All saved journals are listed newest-first in a side panel. Click any entry to preview it, and remove entries cleanly when you want to clear them out.

Design Principles

Why it's built
this way.

01 — Principle
Name the stressor first

Before showing resources, the app asks what you're stressed about. Naming it reduces cognitive load and makes the experience feel personal rather than encyclopedic.

02 — Principle
Local-first, privacy-first

Sensitive writing stays on the student's own machine as plain text. The assistant is the only networked feature, and even there the UI stays minimal and the safety boundaries stay explicit.

03 — Principle
Real help before polished theory

Every screen is built to reduce friction. Real room numbers, direct official links, a bundled PDF, subtopic search, and moderated chat all exist to help a student move one step faster.

Technical

Built with
modern C++ and a small backend.

The desktop app still carries the project, but the current version also includes a lightweight Node/Express service for the assistant, moderation, and crisis-routing logic.

"Clean code is not written by following rules. It is written by someone who cares."

OOP / Classes All screens are QWidget subclasses; MainWindow inherits QMainWindow
Lambdas Signal connections use [this, id]() captures throughout all modules
Signals & Slots Qt event system wires navigation, accordion toggles, journal saves, and clock ticks
HTTP Integration QNetworkAccessManager posts assistant messages to the local backend and renders JSON replies in the chat UI

Backend Safety OpenAI Moderation plus backend keyword checks gate every assistant request before a normal model response is allowed
